# daily_analysis.py Output Format Contract

**Version:** 1.0  
**Last updated:** 2026-07-21

This document specifies the Markdown output format produced by `scripts/daily_analysis.py`. This format is a **versioned contract** parsed by GitHub Pages generators:
- `scripts/generate_history_index.py`
- `scripts/generate_metrics_page.py`

Any changes to this format must maintain backward compatibility or update all dependent parsers.

---

## Document Structure

### 1. Header

```markdown
# Daily Model Analysis — YYYY-MM-DD

**Generated:** <ISO 8601 timestamp>

**Analysis windows:** 7-day (recent), 14-day (medium-term), 28-day (long-term)
```

### 2. Executive Summary

```markdown
## Executive Summary

**Best overall (F-beta=2):** <model_name> @ 7d

**Key findings:**
- <finding 1>
- <finding 2>
...
```

### 3. Model Performance Table (7-day window) — **PARSER CONTRACT**

This table is parsed by `generate_history_index.py` and `generate_metrics_page.py`.

**Format:**
```markdown
## Model Performance (7-day window)

| Model | F1 | Precision | Recall | Status |
|-------|:---:|:---------:|:------:|--------|
| model_name_1         | 0.456 | 0.678 | 0.789 | ✅ |
| model_name_2         | 0.512 | 0.712 | 0.823 | 📊 |
...

**Best overall (F-beta=2):** <model_name> @ 7d
```

**Parser Regex:**
```python
r'\|\s*(\S+)\s*\|\s*([\d.]+)\s*\|\s*([\d.]+)\s*\|\s*([\d.]+)\s*\|'
```

**Constraints:**
- Model names: no whitespace (use underscores)
- Metrics: 3 decimal places (`{:.3f}`)
- Column order: `Model | F1 | Precision | Recall | Status`
- Status column: emoji only (✅ for best, 📊 for others)

**Breaking this format will break GitHub Pages history extraction.**

### 4. Multi-Window Comparison

```markdown
## Multi-Window Comparison

Performance across different time windows...

### F-beta=2 Scores

| Model | 7d | 14d | 28d | Trend |
|-------|:---:|:---:|:---:|:------|
| model_name | 0.623 | 0.645 | 0.678 | 📈 improving |
...

### Precision by Window

| Model | 7d | 14d | 28d |
|-------|:---:|:---:|:---:|
| model_name | 0.678 | 0.698 | 0.712 |
...

### Recall by Window

| Model | 7d | 14d | 28d |
|-------|:---:|:---:|:---:|
| model_name | 0.789 | 0.801 | 0.823 |
...
```

**Trend indicators:**
- `📈 improving`: 28d F-beta > 7d F-beta * 1.1
- `📉 degrading`: 28d F-beta < 7d F-beta * 0.9
- `➡️ stable`: otherwise

### 5. Model Rankings

```markdown
## Model Rankings

### By F-beta=2 (7d, min_precision=0.6)

| Rank | Model | F-beta=2 | Precision | Recall | Notes |
|:----:|-------|:--------:|:---------:|:------:|-------|
| 1 | model_name | 0.734 | 0.712 | 0.823 | ✅ passes filter |
| 2 | model_name | 0.623 | 0.678 | 0.789 | ✅ passes filter |
| 3 | model_name | 0.598 | 0.534 | 0.856 | ❌ low precision |
...

### By F-beta=3 (7d, min_precision=0.6)

Higher recall emphasis (FN≤5% target).

<same structure as F-beta=2>

### By Precision (7d)

For use cases where false positives are costly.

| Rank | Model | Precision | Recall | F1 |
|:----:|-------|:---------:|:------:|:---:|
| 1 | model_name | 0.856 | 0.612 | 0.512 |
...
```

**Ranking logic:**
- F-beta rankings: sort by (passes_filter DESC, fbeta DESC)
- Precision ranking: sort by (precision DESC)
- Show top 5 only

### 6. Precipitation Source Reliability — **PARSER CONTRACT**

```markdown
## Precipitation Source Reliability

Comparison of precipitation sources (7d window):

| Source | Rain Hours | Agreement with Others |
|--------|:----------:|:----------------------|
| OM (Open-Meteo)      |         15 | MS=12h, YX=13h |
| MS (Meteostat)       |         14 | OM=12h, YX=11h |
| YX (Yandex)          |         18 | OM=13h, MS=11h |

**Yandex vs Ground Truth (HA):**
- Yandex rain hours: 18
- Actual rain hours: 16
- Agreement: 14h
- Yandex-only: 4h (false positives)
- Actual-only: 2h (missed events)
```

**Constraints:**
- Source names: `OM (Open-Meteo)`, `MS (Meteostat)`, `YX (Yandex)`
- Rain hours: right-aligned integers
- Agreement: comma-separated `SOURCE=Xh` pairs

### 7. Key Observations & Recommendations

```markdown
## Key Observations & Recommendations

<intelligent analytics based on data>

Examples:

- ⚠️ Best model differs: **model_A** (7d) vs **model_B** (28d). Short-term volatility?
- ⚠️ **Low average precision (0.38)** — models produce many false positives. Pressure-aware variants may help.
- 📈 **ha_live improving** with longer window (7d: 0.623 → 28d: 0.678). Model benefits from more data.

**Next steps:** <specific recommendation>
```
